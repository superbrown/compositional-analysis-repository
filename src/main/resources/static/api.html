data-repository-app REST endpoints

Cateogy Data

api/v02/dataCategories [GET]
api/v02/dataCategories/names [GET]
api/v02/dataCategories/{dataCategoryId} [GET]
api/v02/dataCategories?dataCategoryName={dataCateogryName} [GET]
api/v02/dataCategories/searchableColumnNames?dataCategoryName={dataCateogryName} [GET]

Datasets

api/v02/dataset {JSON in the request body} [POST]
api/v02/datasets [GET]
api/v02/datasets/{datasetId}/sourceDocument [GET] [returns a binary document]
api/v02/datasets/{datasetId}/attachments [GET] [returns a zip file]
api/v02/datasets/{datasetId}/rows [GET]

Dataset Rows

api/v02/rows (request body contains JSON embodying a query) [POST*]
api/v02/rows/flat (request body contains JSON embodying a query) [POST*]
api/v02/rows/asFile (request body contains JSON embodying a query) [POST*] [returns binary document]
api/v02/rows/{rowId} [GET]

Utilities

api/v02/getNamesOfSheetsWithinExcelWorkbook (multipart file in the request body) [POST*]

Admin Operations

NOTE: These have all been made GET operations for ease of execution.
api/v02/addDataCategory?name={dataCateogryName} [GET]
api/v02/removeDataset/{datasetId} [GET]
api/v02/dropDatabaseAndReIngestAllDataFromOriginallyUploadedFiles [GET] [returns JSON with a list of errors that occurred, may be empty]
api/v02/attemptToCleanupDataFromAllPreviouslyIncompleteDatasetUploads [GET] [returns JSON with a list of errors that occurred, may be empty]


_________________________________
* This is logically a GET operation. It is a POST because data must be sent in the requuest body.
